Twl SystemUpdater is a tool used to update DSi development hardware such as IS-TWL-DEBUGGERs and pandas. This does not work on retail units, causing the same -2011 error as TwlNmenu and bricking the console.

TADs in RomFS:

The SystemUpdaters contain the complete development firmware as TAD files for every region in NitroFS. These are sorted into three folders:

NitroFS:/data/debugger/
These TADs are for the IS-TWL-DEBUGGER.
- HNAx, Home menu
- HNBx, System settings (this was removed in v1.2 of the SystemUpdater)

NitroFS:/data/standalone/
These TADs are for the DSi panda unit.
- HNAx, Home menu
- HNBx, System settings (this was removed in v1.2 of the SystemUpdater)

NitroFS:/data/common/
These are TADs that are common between both the IS-TWL-DEBUGGER and panda units.
- HNIx, Nintendo DSi Camera
- HNKx, Nintendo DSi Sound
- HNLx, Version Data
- HNCx, WiFi Firmware
- HNDx, DS Download Play
- HNEx, PictoChat
- HNHx, Nintendo DS Cart Whitelist
- NandFiler
- TwlNmenu

These TAD files have been encrypted with a different key than ones created with maketad.exe, so the only way to get the decrypted SRLs is by running the SystemUpdater and then extracting the contents from NAND.

Mysterious ".nand" file:

There another file in RomFS called menu_launcher.nand (found in NitroFS/data/common/). There has been no mention of this or other .nand files online, though there a reference TWL EVA, which has an option at "Evaluation Menu --> Initializer --> Import *.nand --> menu-lauchner" that makes the .nand file seem like an installable format. menu_launcher.nand has ACSII values for 2435.8325 at the end, which is very similar to the version numbers of the prototype home menu/launcher (1541-6291, and 2235-7985), suggesting that this may be a later revision of that prototype. However, this is currently just speculation.
